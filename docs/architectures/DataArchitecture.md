# 데이터 아키텍처

본 문서는 『랜덤 보디빌딩』 웹 애플리케이션의 데이터 모형, 상태 구조, 데이터 흐름, 영속성 전략 및 제약 조건을 기술한다.

## 데이터 모델

애플리케이션은 크게 세 종류의 정적 데이터 풀과 두 종류의 동적 상태를 다룬다.

### 정적 데이터 (상수)

| 데이터    | 상수명              | 설명                                                              |
| --------- | ------------------- | ----------------------------------------------------------------- |
| 운동 종목 | `DEFAULT_EXERCISES` | 저항운동 리스트 (벤치프레스, 데드리프트, 스쿼트, 컬, 익스텐션 등) |
| 세트      | `DEFAULT_SETS`      | 트라이세트, 컴파운드세트, 슈퍼세트 등                             |
| 포즈      | `DEFAULT_POSES`     | 보디빌딩 규정포즈, 피지크 포즈 등                                 |

#### `DEFAULT_EXERCISES`

운동 종목 풀의 기본 데이터. 저항운동 데이터가 배열로 정의되어 있다. 벤치 프레스, 데드리프트, 로우, 스쿼트, 런지, 컬, 트라이셉스 익스텐션, 어깨 운동, 복부 운동 등 전신을 고루 커버한다.

#### `DEFAULT_SETS`

세트 풀의 기본 데이터. 세트 데이터가 배열로 정의되어 있다. 트라이세트, 컴파운드세트, 슈퍼세트의 세 가지 유형으로 구성된다.

#### `DEFAULT_POSES`

포즈 풀의 기본 데이터. 포즈 모음 데이터가 배열로 정의되어 있다.

### 동적 상태

런타임에 애플리케이션 인스턴스가 관리하는 상태는 풀 객체 내부에 캡슐화되어 있다.

| 상태     | 소속                    | 자료형                                                                     | 설명                                                |
| -------- | ----------------------- | -------------------------------------------------------------------------- | --------------------------------------------------- |
| `items`  | `BasePool` / `PosePool` | `Array<Exercise>` 또는 `Array<ExerciseSet>` 또는 `Array<BodybuildingPose>` | 풀에 들어 있는 항목의 복사본                        |
| `picked` | `BasePool`              | `Record<string, boolean>`                                                  | 중복을 추적하기 위하여 선택된 항목을 객체 키로 관리 |
| `type`   | `BasePool`              | `PoolGubun`                                                                | 풀 구분                                             |

## 영속성〔persistence〕

### 4.1 현재 전략: 메모리 기반

애플리케이션은 모든 상태를 브라우저의 메모리(RAM)에만 보관한다. `localStorage`, `sessionStorage`, `IndexedDB`, 쿠키 등 어떠한 브라우저 저장소도 사용하지 않는다.

```javascript
// 상태가 메모리에만 존재 - 페이지 새로고침 시 모두 손실
this.items = items ? items.slice() : [];
this.picked = {};
```

### 4.2 데이터 손실 시점

| 상황                     | 데이터 손실 여부      |
| ------------------------ | --------------------- |
| 페이지 새로고침 (F5)     | 예 - 모든 상태 초기화 |
| 브라우저 탭 닫기         | 예 - 메모리 해제      |
| 브라우저 재시작          | 예 - 세션 종료        |
| 사용자 목록 편집 후 저장 | 아니오 - 세션 내 유지 |

### 4.3 localStorage 미사용 이유

- **오프라인 우선 설계**: 네트워크가 없어도 동작해야 한다. 저장소 API 호출 자체가 실패할 수 있으므로 최소한의 의존성을 유지한다.
- **상태 단순성**: 추첨 상태는 세션 단위로 충분하다. 사용자가 페이지를 새로고침하면 새로운 운동 루틴을 뽑는 것이 자연스럽다.
- **보안**: 저장소에 민감한 데이터가 없으므로 XSS 공격 표면을 최소화한다.

## 5. 제약 조건 (Constraints)

### 5.1 운동 종목 제약

- **최소 1개**: `saveEditChanges()`에서 `exLines.length === 0` 검사를 통해 운동 종목은 반드시 하나 이상 입력해야 한다.
- **빈 줄 허용 안 함**: `parseTextareaLines()`에서 `line.trim()` 후 `line.length > 0` 필터링을 통해 빈 줄을 제거한다.

```javascript
if (exLines.length === 0) {
    alert("운동 종목을 하나 이상 입력해야 합니다.");
    return;
}
```

### 5.2 세트 제약

- **빈 목록 허용**: 세트 목록은 비워도 된다. `setPool.setItems(setLines)`는 빈 배열도 받아들인다.
- **세트 잔여량 검증**: `validateDraw(true)`에서 `setPool.getRemainingCount() === 0`을 검사하여 세트가 없으면 알림을 띄운다.

### 5.3 포즈 제약

- **빈 목록 허용**: 포즈 목록은 비워도 된다. `PosePool.drawRandom()`는 `items.length === 0`이면 `null`을 반환한다.
- **중복 허용**: 포즈는 복원 샘플링 방식이므로 같은 포즈가 여러 번 뽑힐 수 있다.

### 5.4 추첨 개수 제약

- **최소 1개**: `getDrawCount()`에서 `isNaN(val) || val < 1` 검사를 통해 최소 1개로 제한한다.
- **잔여량 초과 불가**: `validateDraw()`에서 `exercisePool.getRemainingCount() < count` 검사를 통해 남은 종목 수를 초과할 수 없다.

## 6. 데이터 무결성

### 6.1 XSS 방지

모든 사용자 입력 데이터는 화면에 표시되기 전 `escapeHTML()` 함수로 이스케이프 처리된다.

```javascript
var escapeHTML = function (str) {
    return String(str)
        .replace(/&/g, "&amp;")
        .replace(/</g, "&lt;")
        .replace(/>/g, "&gt;")
        .replace(/"/g, "&quot;")
        .replace(/'/g, "&#039;");
};
```

### 6.2 배열 복사

풀 객체는 원본 배열의 복사본을 보관하여 원본 데이터가 의도치 않게 수정되는 것을 방지한다.

```javascript
this.items = items ? items.slice() : [];  // 얕은 복사
```

### 6.3 상태 동기화

`setItems()` 호출 시 `picked` 객체도 함께 초기화하여 항목 목록과 선택 기록이 항상 일관되게 유지된다.

```javascript
BasePool.prototype.setItems = function (newItems) {
    this.items = newItems.slice();
    this.picked = {};  // 풀이 바뀌면 선택 기록도 초기화
};
```
